# Οδηγός Επιβίωσης για Web Servers & Δίκτυα (Go Edition)

Αυτός ο οδηγός δημιουργήθηκε για να εξηγήσει όλες τις βασικές έννοιες των δικτύων, του πρωτοκόλλου HTTP και των servers που χρειάζεσαι για να υλοποιήσεις με επιτυχία το project **ascii-art-web**.

---

## 1. Η Αρχιτεκτονική Client - Server (Πελάτης - Εξυπηρετητής)

Στον παγκόσμιο ιστό, η επικοινωνία βασίζεται σε δύο βασικούς πόλους:

* **Client (Ο Πελάτης / Browser):** Είναι το πρόγραμμα (π.χ. Chrome, Firefox, Safari) που χρησιμοποιεί ο χρήστης για να ζητήσει μια ιστοσελίδα. Λέει στον server: *«Δείξε μου την αρχική σελίδα»*.
* **Server (Ο Εξυπηρετητής / Το Go πρόγραμμά σου):** Είναι ένας υπολογιστής ή ένα πρόγραμμα που τρέχει συνεχώς, "ακούει" για αιτήματα και επιστρέφει τις κατάλληλες απαντήσεις (π.χ. HTML αρχεία, εικόνες, δεδομένα).

> **Σημείωση:** Για το project σου, ο δικός σου υπολογιστής θα εκτελεί ταυτόχρονα τον ρόλο του Client (μέσω του browser σου) και του Server (τρέχοντας τον κώδικα της Go).

---

## 2. IP Διευθύνσεις και Ports (Θύρες)

Για να βρει ο Client πού βρίσκεται ο Server και σε ποιο πρόγραμμα πρέπει να μιλήσει, χρησιμοποιεί δύο στοιχεία:

### Α. IP Διεύθυνση (Το "Σπίτι")
Κάθε συσκευή στο δίκτυο έχει μια μοναδική αριθμητική διεύθυνση. 
* Όταν δουλεύεις τοπικά στον υπολογιστή σου, η διεύθυνσή σου είναι πάντα η **`127.0.0.1`** (γνωστή και ως **`localhost`**). Σημαίνει κυριολεκτικά *"αυτός εδώ ο υπολογιστής"*.

### Β. Port / Θύρα (Το "Διαμέρισμα")
Ένας υπολογιστής μπορεί να τρέχει πολλούς servers ταυτόχρονα. Τα ports (θύρες) είναι αριθμοί από το `1` έως το `65535` που ξεχωρίζουν αυτά τα προγράμματα.
* Το port **`80`** είναι το standard για απλό web (HTTP).
* Το port **`443`** είναι το standard για ασφαλές web (HTTPS).
* Στα δικά μας projects χρησιμοποιούμε custom ports, όπως το **`8080`**.

Συνεπώς, η διεύθυνση για να δοκιμάζεις το site σου θα είναι:  
`http://localhost:8080` ή `http://127.0.0.1:8080`

---

## 3. Το Πρωτόκολλο HTTP (Requests & Responses)

Το **HTTP** (Hypertext Transfer Protocol) είναι η κοινή γλώσσα επικοινωνίας μεταξύ Client και Server. Η επικοινωνία γίνεται πάντα σε ζευγάρια: **Request (Αίτημα)** και **Response (Απάντηση)**.

### HTTP Methods (Μέθοδοι)
Το HTTP χρησιμοποιεί διαφορετικές "ρήματα/μεθόδους" ανάλογα με τον σκοπό του αιτήματος. Στην άσκησή σου χρειάζεσαι δύο:

1.  **`GET` (Ζήτηση Δεδομένων):** Χρησιμοποιείται όταν ο browser θέλει απλώς να κατεβάσει/δείξει κάτι.
    * *Παράδειγμα:* `GET /` -> Ο χρήστης ζητάει την αρχική σελίδα. Ο Server απαντάει στέλνοντας το HTML template της φόρμας.
2.  **`POST` (Αποστολή Δεδομένων):** Χρησιμοποιείται όταν ο browser στέλνει δεδομένα στον server για επεξεργασία (π.χ. δεδομένα φόρμας).
    * *Παράδειγμα:* `POST /ascii-art` -> Ο χρήστης στέλνει το κείμενο που έγραψε και το banner που επέλεξε (π.χ. `shadow`) για να τα επεξεργαστεί η Go.

---

## 4. Endpoints (Διαδρομές) και Handlers στην Go

Όταν το HTTP αίτημα φτάσει στον Go server σου, πρέπει να δρομολογηθεί στο σωστό σημείο.

* **Endpoint / Route:** Είναι το μονοπάτι στο URL (π.χ. `/` ή `/ascii-art`).
* **Handler (Χειριστής):** Είναι η συνάρτηση (function) στην Go που αναλαμβάνει να επεξεργαστεί το συγκεκριμένο endpoint.

Στην Go (χρησιμοποιώντας το standard πακέτο `net/http`), η σύνδεση αυτή γίνεται κάπως έτσι:

```go
// Αντιστοίχιση Endpoints με Handlers
http.HandleFunc("/", homeHandler)          // Διαχειρίζεται το GET /
http.HandleFunc("/ascii-art", asciiHandler) // Διαχειρίζεται το POST /ascii-art

// Εκκίνηση του Server στην πόρτα 8080 (Μπαίνει σε infinite loop και ακούει)
http.ListenAndServe(":8080", nil)

Για να καταλάβουμε γιατί υπάρχουν οι servers και τα πρωτόκολλα, ας σκεφτούμε το internet σαν ένα **παγκόσμιο ταχυδρομικό σύστημα**. Αν δεν υπήρχαν οι servers, το internet δεν θα μπορούσε να λειτουργήσει όπως το ξέρουμε. 

### Γιατί τον χρειαζόμαστε;
Φαντάσου να ήθελες να δεις ένα βίντεο στο YouTube, αλλά αυτό το βίντεο να ήταν αποθηκευμένο στο laptop ενός χρήστη στην Αμερική. Αν αυτός ο χρήστης έκλεινε το laptop του ή αποσυνδεόταν από το ίντερνετ, εσύ δεν θα μπορούσες να δεις το βίντεο. 

Ο **Server** υπάρχει για να λύσει αυτό το πρόβλημα. Είναι ένας υπολογιστής που:
1. Είναι **πάντα ανοιχτός** (24/7/365).
2. Είναι **πάντα συνδεδεμένος** σε πολύ γρήγορο δίκτυο.
3. Είναι **ασφαλής** και κεντρικά ελεγχόμενος.

### Ποια είναι η λειτουργία του;
Η βασική του λειτουργία είναι η **Κεντρική Διαχείριση και Διανομή Πόρων**. 
* **Αποθήκευση:** Κρατάει τα αρχεία (HTML, εικόνες, κώδικα Go, βάσεις δεδομένων).
* **Επεξεργασία (Compute):** Όταν του ζητάς κάτι, δεν σου στέλνει απλά ένα αρχείο. Στην περίπτωσή σου, ο Go server θα πάρει το κείμενο, θα εκτελέσει τον αλγόριθμο ASCII-art, θα "ξοδέψει" επεξεργαστική ισχύ και θα σου επιστρέψει το έτοιμο αποτέλεσμα.
* **Ασφάλεια:** Ελέγχει ποιος έχει πρόσβαση σε τι (π.χ. αν έχεις βάλει σωστό κωδικό).
```

```
## 3. Πόσα Πρωτόκολλα Υπάρχουν και Τι Κάνουν;

Υπάρχουν **εκατοντάδες** πρωτόκολλα, αλλά το internet βασίζεται σε μια συγκεκριμένη "στοίβα" πρωτοκόλλων που ονομάζεται **TCP/IP**. Χωρίζονται σε επίπεδα ανάλογα με τη δουλειά τους. 

Εδώ είναι τα πιο σημαντικά που πρέπει να γνωρίζεις, οργανωμένα ανά κατηγορία:

### Α. Πρωτόκολλα Ιστού και Εφαρμογών (Application Layer)
Αυτά είναι τα πρωτόκολλα με τα οποία έρχεται σε επαφή ο χρήστης και ο κώδικάς σου (όπως η Go).

* **HTTP (Hypertext Transfer Protocol):** Το πρωτόκολλο για τη μεταφορά ιστοσελίδων (HTML, CSS, εικόνες). Λειτουργεί με το μοντέλο Request-Response (αυτό που χρησιμοποιείς στην άσκηση).
* **HTTPS (HTTP Secure):** Η ασφαλής, κρυπτογραφημένη έκδοση του HTTP. Κλειδώνει τα δεδομένα ώστε να μην μπορεί να τα κλέψει κάποιος "ενδιάμεσος" (π.χ. όταν βάζεις την κάρτα σου σε ένα e-shop).
* **FTP (File Transfer Protocol):** Χρησιμοποιείται αποκλειστικά για το ανέβασμα και κατέβασμα μεγάλων αρχείων σε έναν server.
* **DNS (Domain Name System):** Ο "τηλεφωνικός κατάλογος" του internet. Μετατρέπει τα ονόματα (π.χ. `google.com`) στις πραγματικές IP διευθύνσεις των servers (π.χ. `142.250.180.14`).

### Β. Πρωτόκολλα Μεταφοράς Δεδομένων (Transport Layer)
Αυτά λειτουργούν στο παρασκήνιο και αναλαμβάνουν να πάρουν τα δεδομένα του HTTP και να τα σπάσουν σε μικρά κομμάτια (πακέτα) για να τα στείλουν.

* **TCP (Transmission Control Protocol):** Είναι το "αξιόπιστο" πρωτόκολλο. Ελέγχει αν όλα τα πακέτα έφτασαν σωστά και στη σωστή σειρά. Αν ένα κομμάτι χαθεί, ζητάει να ξανασταλεί. 
  * *Πού χρησιμεύει:* Στο Web (HTTP), στα email, στα αρχεία. Αν χανόταν έστω και ένα bit, η ιστοσελίδα ή το ASCII art σου θα παραμορφωνόταν.
* **UDP (User Datagram Protocol):** Είναι το "γρήγορο" πρωτόκολλο. Στέλνει τα πακέτα σφαίρα, χωρίς να τον νοιάζει αν χάθηκε κάποιο στο δρόμο.
  * *Πού χρησιμεύει:* Σε online παιχνίδια (gaming), live streaming και βιντεοκλήσεις. Προτιμάς να κολλήσει το βίντεο για ένα δέκατο του δευτερολέπτου (χάσιμο πακέτου) παρά να καθυστερήσει όλη η ροή.

### Γ. Πρωτόκολλα Δικτύου (Network Layer)
* **IP (Internet Protocol):** Ο υπεύθυνος για τη δρομολόγηση. Βάζει την "ετικέτα" με τη διεύθυνση αποστολέα και παραλήπτη σε κάθε πακέτο δεδομένων, ώστε οι routers να ξέρουν προς τα πού να το στείλουν. (Υπάρχει το κλασικό IPv4, π.χ. `192.168.1.1`, και το νεότερο IPv6).
```

* **`action="/ascii-art"`**: Λέει στον browser σε ποιο endpoint του server να στείλει αυτά τα δεδομένα.
